## Provenance / scope (T17 sizing checkpoint)

This artifact records the STAGED sweep's **sizing checkpoint**, not the
completed full sweep. It was generated by the live driver
(`src.search.solver.fpr_sweep`) over the **first 400** 100%-matched functions
in `build/GALE01/report.json` (`--limit 400`), under the caller-held repo lock,
each dumped via `debug dump local --no-cache-sync`.

- Sizing step ran first, as the spec §5 staged contract requires: of the first
  400 matched functions, **96 bear a class-1 (FPR) COLORGRAPH** (`n_fpr`). The
  smaller `--limit 25` sanity run found 13/25.
- Clean-fixture gate: all 96 parsed without truncation and with complete
  interferer lists (`excluded_truncated: 0`); all 96 are the denominator.
- STRICT classifier (codex major 8 / rev3 §5 "do not relax"): 96/96 G1-perfect
  → `pass` / `fpr_coverage: full`. No miss was relaxed; with zero misses no
  `exclusion_predicates` entry was needed.

The **full unbounded sweep** (all ~18.7k matched functions; ~17.9k resolve to a
local `src/melee/*.c` TU) is the long, **3-working-day time-boxed** operation
per rev3 §5; at ~1.35 s/function it is ~7 h of wall time and was not run to
completion inside this task's budget. Re-running the driver WITHOUT `--limit`
overwrites this doc with the full-sweep record. The fallback ladder is encoded
in `classify_sweep`: any uncharacterized clean miss → `hard_stop`
(remedy `fix-fpr-dispense-reading` ≤5% / `ship-gpr-only` >5%); every miss
covered by a documented static exclusion → `proceed_filtered`
(`fpr_coverage: filtered`).
